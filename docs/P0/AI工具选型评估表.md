# AI 工具选型评估表
## 一、评估说明
本次评估围绕校园互助服务平台（CampusHub）项目开发全流程，对 AI 编程助手、AI 对话工具、AI 代码审查工具、AI 测试工具四类工具进行对比，每类评估 2 个候选工具，确定最终选型。

## 二、工具评估详情

### 1. AI 编程助手
| 评估维度 | 选定工具：Cursor | 未选工具：GitHub Copilot |
| --- | --- | --- |
| 核心功能 | 全项目上下文代码生成、AI 对话、代码重构、错误排查 | 单行/单文件代码补全、代码解释 |
| 选型/不选理由 | 1. 支持全项目上下文理解，适配多模块项目开发<br>2. 内置 AI 对话，无需切换软件，提升编码效率<br>3. 对 CLion、VS Code 兼容友好，团队使用无门槛<br>4. 免费版功能充足，无需教育授权申请 | 1. 部分成员使用 CLion，配置繁琐、兼容性差<br>2. 国内访问不稳定，教育版申请流程复杂<br>3. 仅支持单文件上下文，复杂项目支持不足 |
| 易用性 | 高，开箱即用 | 中，需配置与网络环境 |
| 成本 | 免费版满足项目需求 | 学生需申请教育版，流程繁琐 |
| 团队适配度 | 高 | 低 |

---

### 2. AI 对话工具
| 评估维度 | 选定工具：GPT-4o | 未选工具：文心一言 |
| --- | --- | --- |
| 核心功能 | 需求分析、文档撰写、逻辑推理、代码审查、方案设计 | 日常问答、简单文档生成、基础代码辅助 |
| 选型/不选理由 | 1. 逻辑严谨，适合 IEEE 830 规范文档撰写<br>2. 长文本理解能力强，支持完整需求/架构分析<br>3. 代码理解与问题排查能力强，深度适配开发 | 1. 专业文档生成质量一般，易产生通用套话<br>2. 逻辑推理与代码能力弱于 GPT-4o<br>3. 不满足项目深度开发与设计需求 |
| 易用性 | 高 | 中 |
| 成本 | 免费额度可满足项目使用 | 免费 |
| 团队适配度 | 高 | 中 |

---

### 3. AI 代码审查工具
| 评估维度 | 选定工具：CodeRabbit | 未选工具：通义灵码 |
| --- | --- | --- |
| 核心功能 | GitHub PR 自动审查、代码规范检查、Bug 检测、优化建议 | 编辑器内单文件实时审查、规范提示、简单优化建议 |
| 选型/不选理由 | 1. 可与 GitHub 深度集成，自动审查 PR，契合团队协作流程<br>2. 审查维度全面：规范、性能、安全、可读性<br>3. 支持团队统一质量管控，适合多人协作开发 | 1. 仅支持单文件审查，无 PR 自动化能力<br>2. 审查规则简单，复杂问题检测能力不足<br>3. 无法与 Git 协作流程深度绑定 |
| 易用性 | 中（需配置 GitHub 集成） | 高（开箱即用） |
| 成本 | 免费版满足项目需求 | 免费 |
| 团队适配度 | 高 | 低 |

---

### 4. AI 测试工具
| 评估维度 | 选定工具：Postman AI | 未选工具：Testim AI |
| --- | --- | --- |
| 核心功能 | API 测试用例生成、接口自动化测试、测试脚本生成 | UI 自动化测试、复杂场景测试、视觉验证 |
| 选型/不选理由 | 1. 专注接口测试，完美适配前后端分离架构<br>2. 可一键生成测试用例，操作简单，团队熟悉度高<br>3. 免费版满足项目接口测试需求 | 1. 收费高，学习成本高，不适合学生项目<br>2. 功能冗余，本项目仅需接口测试，无需 UI 自动化<br>3. 配置复杂，短期开发不适用 |
| 易用性 | 高 | 低 |
| 成本 | 免费版够用 | 收费为主 |
| 团队适配度 | 高 | 低 |

## 三、最终选型结论
| 工具类别 | 最终选定工具 |
| --- | --- |
| AI 编程助手 | Cursor |
| AI 对话工具 | GPT-4o |
| AI 代码审查工具 | CodeRabbit |
| AI 测试工具 | Postman AI |
