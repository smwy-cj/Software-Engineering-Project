# AI 协作契约
团队名称：大a再爱我一次
日期：2026年4月13日

第一条：AI 使用原则
- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

第二条：AI 使用范围
- 允许使用 AI 的场景：
  1. 需求文档、团队章程、反思日志等文档初稿生成
  2. 代码片段补全、重复逻辑编写、注释生成、简单函数实现
  3. 测试用例初稿、接口测试脚本辅助编写
  4. 思路梳理、方案讨论、问题排查、代码解释

- 禁止直接使用 AI 输出（需额外审查）的场景：
  1. 核心业务逻辑、订单流程、用户认证等关键代码
  2. 需求最终定稿、架构设计方案、数据库设计
  3. 接口定义、数据结构、系统交互流程
  4. 测试结论、质量评估、阶段验收内容

- 完全禁止使用 AI 的场景：
  1. 直接照搬 AI 输出作为最终交付物，不做任何人工修改
  2. 涉及用户隐私、账号密码、安全认证相关代码
  3. 团队分工、阶段决策、工程判断等管理与决策内容
  4. 抄袭 AI 输出用于学术交付，侵犯原创与规范要求

第三条：过程记录要求
- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思日志》

第四条：代码合入规则
- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数

第五条：违约处理
1. 首次违约：口头提醒，立即重新审查并修正相关内容，补全记录。
2. 二次违约：暂停该成员 AI 使用权限，相关任务全部人工完成。
3. 三次及以上违约：上报课程老师，按课程规定进行处理。
