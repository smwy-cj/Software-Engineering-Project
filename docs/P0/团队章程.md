## 四、AI 工具链选型及理由

### 1. AI 编程助手
- **选定工具：Cursor**
  选型理由：
  1. 支持全项目上下文理解，代码生成、重构、解释一体化，适配校园项目开发场景。
  2. 内置 AI 对话，无需切换软件，编码效率高。
  3. 免费版功能充足，无需申请教育授权，上手快。

- **未选工具：GitHub Copilot**
  不选理由：
  1. 部分成员使用 CLion，Copilot 不够方便。
  2. 复杂项目支持不如 Cursor。

---

### 2. AI 对话工具
- **选定工具：GPT-4o**
  选型理由：
  1. 逻辑严谨、输出规范，适合撰写需求文档、用例建模、架构思路梳理。
  2. 长文本理解与总结能力强，适合 IEEE 830 标准文档生成。
  3. 代码分析、问题排查能力强，能有效辅助开发与讨论。

- **未选工具：文心一言**
  不选理由：
  1. 专业文档生成质量较弱，易出现通用套话，不符合软件工程严谨性要求。
  2. 代码理解与逻辑推理能力不如 GPT-4o，难以满足项目深度协作需求。

---

### 3. AI 代码审查工具
- **选定工具：CodeRabbit**
  选型理由：
  1. 可直接集成 GitHub，自动对 PR 进行代码审查，契合团队 Git 协作流程。
  2. 审查维度全面，包括代码规范、潜在 Bug、性能、可读性与安全问题。
  3. 自动给出优化建议，提升代码质量，适合团队协作开发。
  4. 免费额度可满足课程项目的代码审查需求。

- **未选工具：通义灵码**
  不选理由：
  1. 仅支持编辑器内单文件审查，无 PR 自动化审查能力，不满足团队协作流程。
  2. 审查规则相对简单，复杂代码问题检测能力弱于 CodeRabbit。
  3. 无法与项目 Git 流程深度绑定，不利于团队统一质量管控。

---

### 4. AI 测试工具
- **选定工具：Postman AI**
  选型理由：
  1. 专注接口测试，可自动生成 API 测试用例与脚本，适配前后端分离项目。
  2. 团队熟悉度高，操作简单，支持测试集合管理与批量执行。
  3. 免费版完全满足项目测试需求，使用成本低。

- **未选工具：Testim AI**
  不选理由：
  1. 收费高、配置复杂，远超校园项目需求。
  2. 学习成本高，不适合短期快速开发与测试。 |
