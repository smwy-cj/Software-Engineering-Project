## 用例1：树洞相关操作（参与者：普通用户）
### 1. 用例基本信息
- 用例名称：树洞相关操作
- 参与者：普通用户
- 用例概述：用户匿名发布树洞动态、浏览匿名动态、匿名点赞评论互动，全程保护用户隐私，不暴露用户任何真实个人信息。

### 2. 前置条件
1. 用户已完成**登录平台**操作，账号处于登录状态
2. 用户已完成校内真人学号实名认证，满足平台准入门槛
3. 用户账号状态正常，未被禁言、未被平台封禁
4. 用户已成功进入平台树洞模块页面

### 3. 基本流（正常业务执行流程）
1. 用户进入树洞首页，系统自动加载已审核通过的匿名树洞动态列表，按发布时间倒序展示
2. 用户可浏览动态，支持下拉刷新、上拉加载更多动态；动态仅展示匿名昵称、内容、发布时间、点赞评论数，**不展示用户任何真人信息**
3. 用户点击【发布树洞】按钮，进入匿名动态编辑页面
4. 用户填写文字内容，可选上传图片，选择动态分类，设置是否允许评论、点赞
5. 用户点击【提交发布】，系统首先进行敏感词汇自动检测
6. 内容无违规信息，系统自动生成随机匿名昵称，动态进入人工审核队列
7. 动态审核通过后，自动发布至树洞首页公开列表
8. 用户可对任意树洞动态进行匿名点赞、匿名评论、匿名回复，全程互动均显示匿名昵称
9. 用户可在【我的树洞】页面查看自己发布的全部动态，支持编辑、删除自身动态

### 4. 替代流（异常/分支/失败流程）
替代流1：用户发布内容包含敏感违规词汇、辱骂色情信息
- 系统自动拦截发布请求，弹出提示「内容包含违规敏感信息，请修改后重新发布」，动态不进入审核队列，流程直接结束

替代流2：用户上传图片违规、低俗、非本人实拍图片
- 图片自动审核不通过，系统提示「图片审核失败，禁止发布违规图片」，用户可删除图片重新上传或放弃发布

替代流3：动态人工审核驳回
- 动态不予发布，系统通过消息通知用户具体驳回原因，用户可修改内容后重新提交发布

替代流4：用户账号处于禁言处罚状态
- 系统禁止用户发布动态、评论互动，仅开放浏览权限，弹出提示「您当前处于禁言期，无法发布及互动内容」

### 5. 后置条件
1. 发布成功的树洞动态、评论数据，永久保存至系统数据库
2. 用户发布、点赞、评论操作，实时同步至树洞首页、消息通知模块
3. 用户匿名身份信息全程加密存储，不对其他用户、前台页面泄露
4. 用户退出树洞页面，本次操作结束，后台保留完整操作日志

## 用例2：发布搭子需求（参与者：普通用户）
### 1. 用例基本信息
- 用例名称：发布搭子需求
- 参与者：普通用户
- 用例概述：用户自主发布个性化搭子需求，自定义匹配条件、时间、可见范围；匹配确认前不向对方泄露真人信息，贴合自主发帖、自主选择搭子需求。

### 2. 前置条件
1. 用户已完成**登录平台**操作，账号处于登录状态
2. 用户已完成校内真人实名认证，为在校大学生，排除校外无关人员
3. 用户账号状态正常，无禁言、封禁处罚
4. 用户已进入平台找搭子模块页面

### 3. 基本流（正常业务执行流程）
1. 用户进入找搭子首页，点击【发布搭子需求】，进入需求编辑页面
2. 用户自主填写搭子类型（学习/自习/运动/吃饭/考研/出行等）、需求描述、匹配条件（年级、专业、性别、空余时间、线下地点）
3. 用户设置需求有效期、内容可见范围（同校用户可见/同专业用户可见）
4. 用户开启隐私保护规则：**双方匹配确认前，互相不展示本人真人姓名、学号、联系方式等隐私信息**
5. 用户点击【提交发布】，系统进行敏感词、广告骚扰内容自动审核
6. 内容合规无违规，需求进入人工审核队列
7. 人工审核通过后，需求正式上架搭子首页列表，进入系统智能匹配池
8. 其他用户可自主浏览该需求、发起搭子匹配申请；发布用户可在后台查看全部申请，手动同意/拒绝匹配申请

### 4. 替代流（异常/分支/失败流程）
替代流1：需求内容包含广告推广、恶意骚扰、违规不良信息
- 系统自动拦截提交，弹出提示「内容违规，禁止发布」，流程终止

替代流2：搭子需求人工审核驳回
- 系统发送消息告知用户驳回原因，用户修改需求内容后可重新提交审核

替代流3：用户未填写必填信息（搭子类型、空余时间、需求描述）
- 系统提示「请完善必填搭子信息，方可提交发布」，无法完成提交

替代流4：用户账号处于禁言处罚状态
- 系统禁止用户发布搭子需求，仅可浏览他人搭子信息，弹出权限提示

### 5. 后置条件
1. 审核通过的搭子需求永久保存至数据库，到达有效期后自动下架
2. 需求实时展示在搭子首页，同步进入系统智能匹配池
3. 发布用户可随时编辑、提前下架自身已发布的搭子需求
4. 严格执行隐私规则：匹配双方未确认搭子关系前，互相不可查看真人隐私信息

## 用例3：人工审核（参与者：审核员）
### 1. 用例基本信息
- 用例名称：人工审核
- 参与者：审核员
- 用例概述：审核员对用户发布的树洞动态、搭子需求、交友资料、评论内容进行人工复核，实现敏感词禁用、违规管控、严重违规禁言，保障平台无骚扰无广告健康环境。

### 2. 前置条件
1. 审核员已完成**登录平台**操作，成功进入专属审核后台
2. 系统存在自动审核无法判定、疑似违规的待审核内容队列
3. 审核员拥有合法内容审核操作权限

### 3. 基本流（正常业务执行流程）
1. 审核员进入人工审核后台，系统展示全部待审核内容（树洞动态、搭子需求、恋爱交友内容、用户评论）
2. 审核员逐条查看待审核完整内容，判断是否合规、是否包含敏感违规、广告、恶意骚扰信息
3. 审核员选择对应审核结果：【审核通过】/【审核驳回】/【违规警告】
4. 审核员填写对应审核意见，点击【提交审核结果】
5. 审核通过：内容自动发布至平台前台对应模块
6. 审核驳回：内容退回用户草稿箱，系统推送驳回原因通知用户
7. 违规警告：内容正常发布，同时向用户发送违规提醒，累计严重违规将上报管理员执行禁言处罚
8. 本次审核记录自动归档保存，支持后续追溯查看

### 4. 替代流（异常/分支/失败流程）
替代流1：内容确认严重违规（色情暴力、恶意骚扰、商业广告、造谣诽谤）
- 审核员选择【严重违规驳回】，同时上报管理员后台，对用户执行对应时长禁言处罚，永久留存用户违规记录

替代流2：审核员中途退出审核页面
- 当前未提交的审核操作不保存，已提交审核结果正常生效，流程结束

替代流3：待审核内容已被用户自行提前删除
- 系统自动移除该条待审核数据，审核员无需处理，自动跳转下一条内容

### 5. 后置条件
1. 本次人工审核结果永久保存至系统审核记录库，不可删除篡改
2. 审核结果实时同步至用户端、平台前台内容列表
3. 严重违规内容同步推送至管理员后台，可进行后续禁言、封号处理
4. 完成本次审核操作，审核员可继续处理下一条待审核内容
