# Prompt 日志（P1 需求分析阶段）
## 1. 需求规格说明书生成
**意图**：按 IEEE 830 生成基础需求文档
```text
请为一个校园互助服务平台生成完整的需求规格说明书，平台支持学生发布和接单各类互助需求（快递代取、学习辅导、二手交易、组队匹配等），目标用户为大学生。请按 IEEE 830 标准格式输出。
```

---

## 2. 补充树洞、找搭子、恋爱互助与隐私需求
**意图**：加入特色功能与安全规则
```text
请基于上面的需求规格说明书，加入树洞匿名倾诉、找搭子、恋爱互助功能，强化校内真人认证、隐私保护、无广告无骚扰、敏感词禁用、违规禁言等需求，保持 IEEE 830 格式，并标注需求来源。
```


---

## 3. 用例图角色与用例梳理
**意图**：确定角色与用例清单
```text
请为校园互助平台梳理用例图，至少3个角色、15个用例，包含普通用户、管理员、审核员，覆盖登录、注册、树洞、找搭子、恋爱互助、内容审核、消息通知。
```

---

## 4. DrawIO 用例图绘制指导
**意图**：学习用 DrawIO 画用例图
```text
请一步一步教我用 DrawIO 画校园互助平台的用例图，包括打开、设置、画角色、画用例、连线、导出。
```

---

## 5. 3个核心用例详细描述
**意图**：生成标准用例描述（前置/基本流/替代流/后置）
```text
请根据我的用例图，写3个核心用例的详细描述：树洞相关操作、发布搭子需求、人工审核。格式必须包含：用例基本信息、前置条件、基本流、替代流、后置条件。
```

---

## 6. 用户故事标准化
**意图**：将原始需求转为标准用户故事+验收标准+优先级
```text
请把下面15条用户需求，改写成标准用户故事：作为【角色】，我希望【功能】，以便【价值】。每条必须带验收标准和 MoSCoW 优先级（Must/Should/Could/Won't）。
```
